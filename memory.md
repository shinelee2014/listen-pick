## 📌 1. 本地特有避坑记录

*   **技术坑点/历史 Bug**：
    *   *浏览器极其顽固的 CSS 缓存*：
        *   *Bug描述*：在 Headless Chrome / 常见浏览器中，直接 reload 静态页面，由于静态资源没有配置 Cache-Control，浏览器会极度偏向使用缓存的旧版 `style.css`。
        *   *规程要求*：调试样式或重构排版后，如果发现 computed 属性没有改变，必须在 Console 运行脚本对 `link[rel="stylesheet"]` 的 `href` 加上时间戳 `?v=Date.now()` 强刷，或者做硬性清理。
    *   *月份 Emoji 冲突*：
        *   *Bug描述*：11月 (November) 与 12月 (December) 起初在 Emoji 字典中映射为单数字字符 `1` 和 `2`，这会产生歧义，导致孩子在做 11/12 月题目时错误点击 1/2 月份卡片。
        *   *规程要求*：月份 Emoji 应精确对照，11月使用双键 `1️⃣1️⃣`，12月使用双键 `1️⃣2️⃣`。
    *   *iPad 1024x768 视口高度溢出*：
        *   *Bug描述*：在 iPad 设备的 landscape 视口中，由于高度只有 768px，2x2 (4选项) 的卡片网格配合上方较大的喇叭播放按钮和标题，容易被挤出底边，导致出现丑陋的滚动条。
        *   *规程要求*：必须配合 `@media (max-height: 800px) and (min-width: 768px)` 特殊微调。将 4选项网格的最大宽度限制在 `340px`，并将大喇叭缩小为 `56px` 宽，高度无缝贴合单屏。

## ⚖️ 2. 重大技术决策日志 (ADR)

*   **[2026-06-26] 使用纯 Emoji 作为选项载体**：
    *   *背景痛点*：听音选图本需 600 张精美插图，若使用生图模型会极大消耗 LLM API Token，且离线状态下静态加载 600 张图片会让应用体积暴增，影响首屏响应速度。
    *   *最终选型*：使用浏览器系统内置的 Emoji 集合，通过自定义词汇映射表对英文句子内的名词/节日/动作进行高保真翻译，全本地化加载。
    *   *影响约束*：在新增句子或单词时，必须在 `js/app.js` 的 `getEmojiForImage()` 中添加对应的词汇 -> Emoji 映射，否则会降级显示为备用图片。

*   **[2026-06-27] 免密多孩子账户本地 localStorage 隔离**：
    *   *背景痛点*：家庭多子女共用一个 iPad/浏览器，通关进度会产生交叉混淆，需要能分别建档并随时一键重置。
    *   *最终选型*：采用完全前端的 localStorage 前缀化命名机制（形如 `listen_pick_progress_[姓名]`）。
    *   *影响约束*：所有的数据存储功能全部委托给 `Storage` 模块统一代理，主逻辑（`app.js`）完全解耦，不直接调用原始 `localStorage`。

## 📝 3. 关键业务线与接口备忘

*   **核心 API / 函数表**：
    *   `Storage.getGradeCompletion(grade, totalStages)`：计算某个年级的练习完成率（以 100 关为基数）。
    *   `Storage.resetCurrentChild()`：只删除当前正在登录的这个孩子的 localStorage 数据。
    *   `AudioPlayer.play(sentence, audioFile, speed)`：发音服务。优先尝试本地 MP3，如空，则使用原生 `speechSynthesis` 进行高音质发音。
    *   `createStarBurst(sourceElement)`：利用 JS 计算屏幕绝对坐标，让获得的星星从卡片精确飞往右上角状态栏，并在落地时让状态栏震动反馈。
    *   `createConfetti()`：利用全局的 `canvas-confetti` 库触发高品质多色左右喷泉粒子洒花。
